# Resource Hacker AI 翻译器使用说明

## 1. 工具用途

`ResourceHackerAITranslator.exe` 是 Resource Hacker 的辅助翻译工具。它不会修改 Resource Hacker 原程序本体，而是通过剪贴板和按键自动化读取 Resource Hacker 当前编辑区内容，调用 AI 翻译接口，再把译文回传到编辑区。

适合用于：

- 翻译 Resource Hacker 中的菜单、对话框、字符串表等资源脚本。
- 保持控件 ID、位置、大小、样式、快捷键等结构不变，只汉化可见文本。
- 在翻译后预览、手动修正译文，再回传到 Resource Hacker 编译。

## 2. 文件组成

推荐把整个 `dist` 目录复制到使用电脑：

- `ResourceHackerAITranslator.exe`：主程序。
- `translator.config.json`：翻译接口配置文件。

如果移动到其他电脑，至少需要保证这两个文件在同一个目录。

注意：配置文件中的 API Key 是明文保存。发给别人前请删除或替换密钥。

## 3. 运行条件

- Windows 系统。
- 安装或自带 .NET Framework 4.x。
- 已安装并打开 Resource Hacker。
- 电脑可以访问所配置的翻译 API。
- Resource Hacker 窗口标题中应包含 `Resource Hacker`。

## 4. 启动方法

双击运行：

```text
ResourceHackerAITranslator.exe
```

启动后会出现主窗口，并在检测到 Resource Hacker 窗口时显示悬浮栏。

## 5. API 配置说明

主窗口上方可以选择翻译接口，并填写接口参数。

常见字段：

- 接口：选择 `openai`、`deepseek`、`doubao`、`qwen`、`microsoft`、`google`、`youdao` 或自定义兼容接口。
- 目标语言：默认 `Simplified Chinese`，可下拉选择，也可自定义输入。
- Endpoint：接口地址。
- Model：模型名称，可下拉选择，也可自定义输入。
- API Key：接口密钥，默认显示为 `*`，点击“显示”可临时查看。
- App Key：部分接口使用，默认显示为 `*`，点击“显示”可临时查看。
- App Secret：部分接口使用。
- Region：微软等接口可能需要。
- 源语言：默认 `auto`，可下拉选择，也可自定义输入。
- 目标代码：默认 `zh-Hans`，可下拉选择，也可自定义输入。

填写后点击“保存配置”。

可点击“测试接口”发送一段测试翻译，确认接口可用。

## 6. 主窗口按钮说明

- 翻译全部：自动全选 Resource Hacker 当前编辑区内容，复制、翻译并回传。
- 翻译选中：只翻译 Resource Hacker 当前选中的文本，并把译文替换选区。
- 保存配置：保存当前 API 设置到 `translator.config.json`。
- 测试接口：测试当前翻译接口是否可用。
- 回传译文：把右侧“译文”视图中的内容复制并粘贴回 Resource Hacker 编辑区。
- 撤销回传：撤销最近一次回传或翻译粘贴。
- 悬浮栏：显示或隐藏悬浮工具栏。
- 打开配置：用记事本打开配置文件。

点击主窗口按钮后，工具可能会临时切换到 Resource Hacker 完成复制/粘贴。操作结束后，主窗口会恢复到原来的位置、大小和显示状态。

## 7. 悬浮栏按钮说明

悬浮栏会吸附在 Resource Hacker 窗口附近，方便直接操作。

- 全选：全选当前 Resource Hacker 编辑区。
- 复制：复制当前 Resource Hacker 编辑区内容。
- 全译：翻译整个当前编辑区并回传。
- 选译：翻译当前选中内容并替换选区。
- 回传译文：把主窗口右侧译文回传到 Resource Hacker。
- 撤销回传：撤销最近一次回传或选译粘贴。
- 主窗：显示主窗口。
- 复制译文：把主窗口右侧译文复制到剪贴板。
- 粘贴译文：把剪贴板中的译文粘贴到 Resource Hacker。

## 8. 翻译结果视图

主窗口下方“翻译结果”页包含：

- 左侧：原文。
- 右侧：译文。
- 两侧都有自动行号。

译文框可编辑。建议在回传前检查：

- 行数是否一致。
- 控件 ID 是否保留。
- 坐标、大小、样式是否未被翻译。
- `CAPTION`、`MENUITEM`、`LTEXT` 等语法结构是否仍可编译。

如果原文和译文行数不同，日志会出现警告：

```text
WARNING: line count changed.
```

看到该提示时，应先人工检查译文，再编译资源。

## 9. 撤销说明

“撤销回传”用于防止译文错误导致 Resource Hacker 编译失败。

- 全译 / 回传译文：工具会保存回传前的完整编辑区内容，撤销时恢复全文。
- 选译 / 翻译选中：工具会使用 Resource Hacker 编辑器自身的撤销操作撤回刚才的选区替换，避免误覆盖全文。

建议每次发现译文格式异常或编译失败时，立即点击“撤销回传”。

## 10. 热键

- `Ctrl + Alt + T`：翻译当前完整编辑区。
- `Ctrl + Alt + Shift + T`：翻译当前选中内容。

使用热键前，请确保焦点在 Resource Hacker 的编辑区。

## 11. 常见问题

### 提示 No text was copied

原因通常是焦点不在 Resource Hacker 编辑区。

处理方法：

1. 点击 Resource Hacker 的脚本编辑区。
2. 确认能手动 `Ctrl+C` 复制内容。
3. 再点击翻译按钮。

### 译文行数变化

说明 AI 返回的文本结构和原文不一致。请检查译文视图，必要时手动修正，保持一行对应一行。

### 其他电脑无法使用

请确认：

- EXE 和 `translator.config.json` 在同一目录。
- 系统有 .NET Framework 4.x。
- API 配置正确。
- Resource Hacker 已打开。

### 密钥是否安全

界面中 `API Key` 和 `App Key` 默认用 `*` 隐藏，但 `translator.config.json` 内仍是明文。不要把带真实密钥的配置文件发给别人。
