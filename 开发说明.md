# Resource Hacker AI 翻译器开发说明

## 1. 项目定位

本项目是 Resource Hacker 的外部辅助翻译桥接程序，不直接逆向修改或 patch Resource Hacker 主程序。

核心思路：

1. 通过窗口查找定位 Resource Hacker。
2. 使用正常键盘操作和剪贴板读取当前编辑区内容。
3. 调用配置的翻译接口。
4. 在主窗口中展示原文和译文。
5. 根据用户操作将译文粘贴回 Resource Hacker 编辑区。

这样做的优点是实现风险低、便于移植，也不破坏 Resource Hacker 原始文件。

## 2. 主要文件

- `TranslatorBridge.cs`：主程序源码，WinForms 单文件实现。
- `translator.config.json`：翻译接口配置模板。
- `Build-Exe.cmd`：使用 .NET Framework C# 编译器打包 EXE。
- `Test-ConfigAndExe.ps1`：基础 smoke test。
- `Launch-Translator.cmd`：启动辅助脚本。
- `Start-ResourceHackerAITranslator.ps1`：启动脚本。
- `dist\ResourceHackerAITranslator.exe`：构建输出。

## 3. 构建方法

在项目目录执行：

```powershell
cd E:\ai-translate-bridge
.\Build-Exe.cmd
```

如果 EXE 正在运行，构建前应先关闭：

```powershell
Get-Process ResourceHackerAITranslator -ErrorAction SilentlyContinue | Stop-Process -Force
```

构建成功后输出：

```text
dist\ResourceHackerAITranslator.exe
```

## 4. 测试方法

执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\Test-ConfigAndExe.ps1
```

当前已知正常 warning：

```text
ProviderConfig.type 从未赋值
ProviderConfig.prompt 从未赋值
```

这是 JSON 配置兼容字段导致，不影响运行。

## 5. 程序结构

主要类型：

- `Program`：WinForms 入口。
- `MainForm`：主窗口、配置 UI、翻译流程、回传和撤销逻辑。
- `FloatingToolbar`：吸附在 Resource Hacker 附近的悬浮工具栏。
- `NativeMethods`：Win32 窗口查找和激活。
- `Translator`：不同翻译接口的调用入口。
- `AppConfig` / `ProviderConfig`：配置文件模型。
- `TextShape`：文本结构辅助，例如行数统计。
- `HotKeyWindow`：全局热键注册。

## 6. 主窗口 UI 说明

主窗口使用 `TableLayoutPanel` 组织：

- 顶部说明文本。
- API 配置区域。
- 操作按钮区域。
- 下方 Tab 页：
  - 翻译结果。
  - 日志。

配置区中：

- `providerBox` 是只读下拉。
- `targetLanguageBox`、`modelBox`、`sourceBox`、`targetBox` 是可输入下拉框。
- `apiKeyBox`、`appKeyBox` 默认使用 `UseSystemPasswordChar = true` 隐藏内容，并通过“显示/隐藏”按钮切换。

翻译结果区中：

- `sourcePreviewBox` 显示原文。
- `resultPreviewBox` 显示译文。
- `sourceLineNumberBox` 和 `resultLineNumberBox` 显示行号。

## 7. 悬浮栏说明

`FloatingToolbar` 是一个 `Form`，带有：

- `TopMost = true`
- `ShowInTaskbar = false`
- `WS_EX_NOACTIVATE`
- `WS_EX_TOOLWINDOW`

它通过 `UpdateFloatingToolbar()` 定时跟随 Resource Hacker 窗口。

默认位置由 `GetDefaultFloatingLocation()` 计算，目前基于 Resource Hacker 窗口左上角偏移。

## 8. 翻译流程

完整翻译流程在 `TranslateFocusedEditor(bool onlySelection)` 中：

1. 保存当前 UI 配置。
2. 激活 Resource Hacker。
3. 如果是全译，发送 `Ctrl+A`。
4. 发送 `Ctrl+C` 复制文本。
5. 从剪贴板读取原文。
6. 调用 `Translator.Translate(config, source)`。
7. 更新原文/译文预览和行号。
8. 检查行数是否变化。
9. 保存撤销状态。
10. 写入剪贴板并发送 `Ctrl+V` 回传。

## 9. 回传和撤销逻辑

回传相关方法：

- `PasteResultToFocusedEditor()`
- `PasteTranslationToFocusedEditor(bool replacePreviousFullCopy)`
- `CaptureCurrentEditorForUndo()`
- `UndoLastPaste()`

撤销分两种模式：

1. 全文恢复模式

   用于全译和手动回传译文。回传前保存 Resource Hacker 当前完整编辑区内容到 `lastBeforePasteText`。撤销时全选并粘贴旧内容。

2. 编辑器撤销模式

   用于翻译选中。选译后设置 `lastUndoUsesEditorUndo = true`。撤销时向 Resource Hacker 发送 `Ctrl+Z`，避免把整个编辑区覆盖掉。

## 10. 主窗口状态保持

后台任务使用 `RunBusy()` 启动。

任务开始前调用：

```csharp
CaptureMainWindowSnapshot()
```

记录主窗口：

- 是否可见。
- 窗口状态。
- 位置和大小。

任务结束后调用：

```csharp
RestoreMainWindowSnapshot(...)
```

如果任务是从主窗口触发，操作完成后会恢复主窗口原位置、原大小和前台显示。

## 11. 翻译提示词要求

翻译 Resource Hacker 脚本时，提示词必须强调：

- 只翻译可见文本。
- 保留控件 ID。
- 保留位置和大小。
- 保留样式、宏、数字、资源 ID。
- 保留快捷键标记，例如 `&File` 中的 `&`。
- 保留引号、转义、花括号、缩进和空行。
- 一行输入对应一行输出。
- 不输出 Markdown，不解释。

如果模型输出行数变化，程序只做警告，不会自动修复。后续可考虑增加结构化校验或自动回滚策略。

## 12. 配置文件说明

`translator.config.json` 包含：

- 顶层 `provider`：当前接口名称。
- 顶层 `targetLanguage`：目标语言。
- `providers`：各接口配置。

常用字段：

- `endpoint`
- `model`
- `apiKey`
- `appKey`
- `appSecret`
- `region`
- `source`
- `target`
- `prompt`

注意：密钥字段在配置文件中是明文。UI 隐藏只防止界面暴露，不能保护配置文件本身。

## 13. 编码注意事项

历史上源码曾出现中文乱码。维护时建议：

- C# 源码中的中文 UI 文本优先使用 `\uXXXX` 转义。
- 避免用错误编码的 PowerShell 写入源码。
- 修改后必须重新编译并运行 smoke test。

## 14. 发布到其他电脑

发布时建议复制整个目录：

```text
dist\
```

至少包含：

- `ResourceHackerAITranslator.exe`
- `translator.config.json`

如果不想泄露密钥，发布前清空配置文件中的：

- `apiKey`
- `appKey`
- `appSecret`

## 15. 后续可改进方向

- 增加 App Secret 的显示/隐藏按钮。
- 增加配置导入/导出功能。
- 增加 Resource Hacker 编辑区控件级定位，减少对剪贴板和 SendKeys 的依赖。
- 增加译文结构校验，例如资源 ID、坐标、样式是否被改动。
- 增加自动备份 `.rc` 或 Resource Hacker 当前脚本内容到本地文件。
- 增加多语言 UI 资源文件，避免硬编码文本。
